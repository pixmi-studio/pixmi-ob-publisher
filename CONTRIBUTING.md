# Contributing to Pixmi Obsidian WeChat Publisher

Thank you for your interest in contributing to Pixmi Obsidian WeChat Publisher! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs

- Check the [Issues](https://github.com/pixmi-studio/pixmi-ob-publisher/issues) to see if the bug has already been reported.
- If not, create a new issue using the bug report template.
- Provide as much detail as possible, including steps to reproduce and logs from the plugin's settings tab.

### Suggesting Enhancements

- Open a new issue with a clear title and description of the suggested enhancement.
- Explain why this feature would be useful to other users.

### Pull Requests

1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name` or `fix/your-fix-name`.
3. Make your changes and ensure the code follows the existing style.
4. Run tests to make sure everything is working: `npm run test`.
5. Commit your changes: `git commit -m "feat: add some feature"`.
6. Push to the branch: `git push origin feature/your-feature-name`.
7. Open a Pull Request.

## Development Setup

See the [Development](#development) section in the README.md for setup instructions.

# 贡献指南

感谢你对 Pixmi Obsidian WeChat Publisher 的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告 Bug

- 在 [Issues](https://github.com/pixmi-studio/pixmi-ob-publisher/issues) 中查看该 Bug 是否已被报告。
- 如果没有，请使用 Bug 报告模板创建一个新 Issue。
- 请提供尽可能详尽的信息，包括复现步骤和插件设置面板中的日志。

### 提交功能建议

- 创建一个新 Issue，提供清晰的标题和功能描述。
- 说明为什么这个功能对其他用户也有帮助。

### 提交代码 (Pull Requests)

1. Fork 本仓库。
2. 创建新分支：`git checkout -b feature/your-feature-name` 或 `fix/your-fix-name`。
3. 进行开发，并确保代码风格与现有代码一致。
4. 运行测试：`npm run test`。
5. 提交更改：`git commit -m "feat: add some feature"`。
6. 推送分支：`git push origin feature/your-feature-name`。
7. 开启一个 Pull Request。

## 开发环境搭建

请参考 README.md 中的 [Development](#development) 章节。
