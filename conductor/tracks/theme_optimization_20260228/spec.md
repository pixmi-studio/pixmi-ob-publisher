# Specification: `theme_optimization_20260228`

## Overview
优化上传至微信草稿箱后的文档排版格式，重点改进 `medium geek` 主题，解决换行丢失、移动端阅读体验、中英文混合排版及配色方案生硬等问题。

## Functional Requirements
1. **换行逻辑修复**：
   - 修复上传后文章丢失换行的问题，确保 Markdown 换行符（回车或双空格）在微信编辑器中正确渲染为段落间距。
2. **移动端阅读优化 (`medium geek`)**：
   - **内边距 (Padding)**：增加左右 padding（建议 15px-20px），以适配小屏幕设备。
   - **字体大小 (Font Size)**：整体调小字体（建议正文 14px-15px），并同步缩小各级标题（H1-H6）的大小，保持比例协调。
   - **字间距 (Letter Spacing)**：优化数字、英文及标点符号的字间距。
3. **中英文混合排版优化**：
   - 优化 `word-break` 逻辑，确保长单词或 URL 在移动端不会导致容器溢出或异常截断。
   - 优化中文全角标点符号与周围文字的间距。
4. **视觉配色优化 (`Minimalist Geek` 方案)**：
   - **引用块 (Blockquote)**：采用深灰色/灰蓝色调，去除生硬感。
   - **代码块 (Code Block)**：优化背景色与边框，确保在微信暗色模式下也有较好兼容性。
   - **整体配色**：降低色彩饱和度，提升长文阅读的舒适度。

## Non-Functional Requirements
- **兼容性**：确保优化后的 CSS 样式在微信内置浏览器及不同版本的微信草稿箱编辑器中均能生效。
- **可维护性**：优化 `src/themes.ts` 和相关 CSS 转换逻辑，便于后续扩展其他主题。

## Acceptance Criteria
- [ ] 文章上传后，段落间有明显的视觉换行间隔。
- [ ] `medium geek` 主题在手机端的阅读体验有显著提升（文字不贴边，字体大小适中）。
- [ ] 中英文混合排版时，标点及单词换行自然，无布局破碎感。
- [ ] 引用块、代码块配色符合“极简极客”风格，不再生硬。

## Out of Scope
- 其他主题（如 `default` 等）的大规模重构。
- 对 Obsidian 内部预览模式的独立排版调整（仅针对上传至微信的逻辑）。
